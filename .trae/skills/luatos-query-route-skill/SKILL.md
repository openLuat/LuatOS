# LuatOS查询路由技能

## 概述

本技能用于处理用户关于合宙产品和LuatOS开发的所有咨询问题，包括但不限于：
- 合宙模组/开发板的使用问题
- LuatOS API函数调用
- 代码错误调试
- 硬件接口编程
- 文档查询
- 产品方案选型推荐

## 触发条件判断

**必须调用本技能的场景：**
- 用户询问合宙产品（如Air780EPM、Air8000W、Air8101等）
- 用户询问LuatOS开发相关问题
- 用户询问API用法、库函数
- 用户报告代码报错、运行异常
- 用户需要产品方案选型推荐
- 用户需要代码生成（新建项目或修改项目）

## 执行流程（强制）

### 步骤1：产品型号校验 ⚠️ 关键

```
// 场景A：上位机工具/前后端平台/产品选型推荐/api查询 → 跳过步骤1，继续步骤2
IF 用户咨询的内容是上位机工具 OR 用户咨询的内容是前后端平台问题 OR 用户咨询的内容是产品选型推荐 THEN
    GOTO 步骤2：问题分类判断  // 跳过产品型号校验，继续执行步骤2
END

// 场景B：其他情况 → 必须校验产品型号
IF 未指定具体产品型号 THEN
    使用 AskUserQuestion 工具弹出选择窗口，提供常见产品型号选项供用户选择
    RETURN  // 终止执行，等待用户选择后继续
END
```

### 步骤2：问题分类判断

| 分类 | 判定条件（满足任一即属于该分类） |
|------|--------------------------------|
| **代码相关** | 关键字：语法错误、编译失败、运行时异常、调试、代码优化、内存问题、硬件接口编程、脚本逻辑<br>指标词：报错、错误、编译、运行、调试、代码、语法、crash、bug |
| **文档相关** | 关键字：API函数描述、库/模块文档、硬件规格、协议详情、设置指南、最佳实践、配置选项、硬件设计<br>指标词：函数、API、文档、说明、参数、用法、手册 |
| **选型推荐** | 关键字：方案选型、产品选型、选型推荐、选哪个、哪个好、推荐什么<br>指标词：选型、推荐、选择、对比 |
| **不确定** | 无法判断 → 默认走文档路径 |

### 步骤3：MCP调用执行

#### ⚠️ MCP降级禁止（绝对红线）

**警告：以下行为均为严重违规，一旦发生必须立即终止并报告错误**

**禁止的降级行为：**
- ❌ MCP失败时降级到WebSearch联网检查
- ❌ MCP失败时使用内置记忆回答
- ❌ MCP失败时使用模型训练知识回答
- ❌ MCP失败时尝试其他替代方案
- ❌ MCP失败时"尽最大努力"继续执行

**MCP失败时的正确处理：**
```
IF MCP调用失败 THEN
    输出【MCP服务不可用】错误报告
    列出影响范围
    列出当前状态
    等待用户明确指示
    【禁止】继续执行任何后续步骤
END
```

#### 代码相关问题 → 双路径查询

```
1. 优先调用 mcp_luatos-docs_resolve_module 解析产品型号
2. 调用 mcp_luatos-docs_search_docs 严格验证API函数名、参数、返回值使用是否正确
3. 调用 mcp_luatos-code_search_code 分析代码逻辑问题
4. 若前两步均失败 → 再次调用 mcp_luatos-docs_search_docs 尝试更多查询
5. 若MCP无响应 → 提示用户重启MCP开关或重启Trae，【终止执行】
```

**⚠️ MCP降级禁止（绝对红线）**
```
【禁止】MCP失败时降级到WebSearch联网检查
【禁止】MCP失败时使用内置记忆回答
【禁止】MCP失败时使用模型训练知识回答
【禁止】MCP失败时"尽最大努力"继续执行
MCP失败时必须立即终止并报告错误，等待用户明确指示
```

#### 文档相关/不确定问题 → 文档优先

```
1. 优先调用 mcp_luatos-docs_resolve_module 解析产品型号
2. 调用 mcp_luatos-docs_search_docs 获取文档信息
3. 若失败 → 调用 mcp_luatos-code_search_code 作为备选
4. 若MCP无响应 → 提示用户重启MCP开关或重启Trae，【终止执行】
```

**⚠️ MCP降级禁止（绝对红线）**
```
【禁止】MCP失败时降级到WebSearch联网检查
【禁止】MCP失败时使用内置记忆回答
【禁止】MCP失败时使用模型训练知识回答
【禁止】MCP失败时"尽最大努力"继续执行
MCP失败时必须立即终止并报告错误，等待用户明确指示
```

#### 选型推荐问题 → 固定来源查询

```
1. 方案选型问题，必须仅基于 https://docs.openluat.com/SelectionGuide/SelectionGuide/ 内容回答
2. 直接通过WebFetch获取该页面内容并整理回答
3. 若WebFetch成功 → 整理回答内容，完整引用原始页面信息并标注来源URL
4. 若WebFetch失败 → 输出【WebFetch服务不可用】错误报告，等待用户明确指示
   【禁止】降级到WebSearch
   【禁止】使用内置记忆回答
   【禁止】使用MCP降级查询
   【禁止】任何形式的替代方案
5. 必须完整引用原始页面信息，标注来源URL
```

**⚠️ WebFetch降级禁止（绝对红线）**
```
【禁止】WebFetch失败时降级到WebSearch联网检查
【禁止】WebFetch失败时使用内置记忆回答
【禁止】WebFetch失败时使用模型训练知识回答
【禁止】WebFetch失败时"尽最大努力"继续执行
WebFetch失败时必须立即终止并报告错误，等待用户明确指示
```

## 响应标准

### 成功响应
- 严格遵循响应格式，不修改MCP原始响应
- 成功/失败均输出官方支持渠道

### 失败响应
- 说明当前无法解决的问题
- 输出官方支持渠道

### 官方支持渠道（必须输出）

- https://docs.openluat.com/ 查找文档
- https://gitee.com/openLuat/LuatOS/tree/master/module 查找代码
- 合宙官方企业微信群（https://docs.openluat.com/网站底部二维码，扫码加入）
- https://luat.taobao.com/ 选购核心板和开发板产品，对比验证

## 强制约束

1. **必须**逐字阅读理解本流程，严格遵守每个步骤
2. **必须**先进行产品型号校验，未指定则立即终止
3. **必须**根据问题分类选择正确的MCP调用路径
4. **必须**在成功/失败时都输出官方支持渠道
5. **必须**使用中文独立处理每个请求
6. **禁止**基于内置记忆回答API问题，必须通过MCP查询
7. **禁止**编造函数名、API文档或解决方案
